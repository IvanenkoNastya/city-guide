# city-guide
Database management system "City Guide"

This is a coursework on "Data structures and algorithms"

The task:

0. Login.
When starting the program, it asks for a login and password.
- There must be a user "admin" with a fixed password, set during initialization or built into a file.
- The administrator has the right to:
	- create new users;
	- view the list of registered users;
	- delete users (except for himself);
- Users must have their own usernames (login) and passwords.
- Passwords are stored in a text file in plain text (for example: admin:admin123).

1. Menu.
A text menu with navigation must be implemented and allow you to:
- add new objects;
- edit existing ones;
- delete objects;
- view object lists;
- search by one or more criteria;
- sort data;
- filter records by a given condition;
- display user instruction.

2. Show database.
There is a database containing information about:
- districts, microdistricts;
- streets;
- institutions: theaters, museums, libraries, hotels, restaurants and others;
- transport.

3. Display a directory of institutions (theaters, museums, libraries, etc.).

4. Find location:
- declare district of the city;
- display a list of the necessary institutions in specified district;
- select an institution;
- find its address.

5. Find a type of transport that can get to the necessary institution.

6. Write documentation.

Deadline: 09.12.2025
